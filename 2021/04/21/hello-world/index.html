Markdown介绍

Markdown是一种轻量级标记语言，排版语法简洁，让人们更多的关注内容本身而非排版。它使用易读易写的纯文本格式编写文档，可与HTML混编，可导出HTML、PDF以及本身的.md格式的文件。因简洁、高效、易读、易写，Markdown被大量使用，如Github、Wikipedia等网站，如各大博客平台:WordPress、Drupal、简书等。

Markdown基本语法

标题语法

Markdown通过在单词或者语句前加#号来创建标题。#的数量表示了该标题的级别。#号的数量越多，则该标题的级别越低，其在网页端显示的也越小。

不同的Markdown应用程序处理#号和标题之间的空格方式并不一致，为了兼容考虑，最好用一个空格在#和标题之间进行分隔。例如我的当前所处的标题是标题语法，我应该写成### 标题语法比较好。

段落语法

要创建段落，使用空白行将一行或者多行文本进行分隔即可。

不要用空格（space）或者制表符（tabs）缩进段落。

换行语法

在一行的末尾添加两个或多个空格，然后按回车键,即可创建一个换行(<br>)。

几乎每个 Markdown 应用程序都支持两个或多个空格进行换行，称为 结尾空格（trailing whitespace） 的方式，但这是有争议的，因为很难在编辑器中直接看到空格，并且很多人在每个句子后面都会有意或无意地添加两个空格。由于这个原因，你可能要使用除结尾空格以外的其它方式来换行。幸运的是，几乎每个 Markdown 应用程序都支持另一种换行方式：HTML 的 <br> 标签。

为了兼容性，请在行尾添加“结尾空格”或 HTML 的 <br> 标签来实现换行。

还有两种其他方式我并不推荐使用。CommonMark 和其它几种轻量级标记语言支持在行尾添加反斜杠 () 的方式实现换行，但是并非所有 Markdown 应用程序都支持此种方式，因此从兼容性的角度来看，不推荐使用。并且至少有两种轻量级标记语言支持无须在行尾添加任何内容，只须键入回车键（return）即可实现换行。
